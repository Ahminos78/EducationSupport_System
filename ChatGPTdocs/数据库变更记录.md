# 数据库变更记录

记录时间：2026-07-18

## 变更说明

数据库从第一阶段（仅用户 + 课程 + 选课 + 作业 + 考试 + 讨论）扩展为支持教学班、排课、成绩组成和学业预警。

## 表总览

### 保留并修改的表

| 表 | 变更 |
|---|---|
| tb_course | 新增 academic_year, semester, total_hours 三列 |
| tb_enrollment | course_id 改为 class_id（引用 tb_course_class），新增 course_id 冗余列、final_score、grade_letter、passed |

### 新增的表

| 表 | 位置（init.sql 行号） | 说明 |
|---|---|---|
| tb_course_class | 行 116 | 教学班，每个课程可多个，每个配一位教师 |
| tb_course_schedule | 行 134 | 排课信息：星期、节次、教学周、教室 |
| tb_grade_component | 行 180 | 成绩组成项：每门课定义权重和评分项 |
| tb_student_grade | 行 195 | 学生成绩明细：每人每项的得分 |
| tb_academic_warning | 文件末尾 | 学业预警记录 |

### 删除的表

| 表 | 原因 |
|---|---|
| tb_course_selection | tb_enrollment 前身，无数据 |
| tb_forum_post | 旧论坛，被 tb_discussion 取代 |
| tb_forum_reply | 旧论坛，被 tb_discussion 取代 |
| tb_discussion + edu-interaction-service | 讨论功能当前阶段阉割 |

## 已有数据库执行顺序

```bash
# 1. 清理废弃表
mysql -uroot -p edu_platform < sql/migrate_cleanup_obsolete_tables.sql

# 2. 第一批：课程扩展 + 教学班
mysql -uroot -p edu_platform < sql/migrate_course_extend.sql

# 3. 第三批：排课 + 选课改造
mysql -uroot -p edu_platform < sql/migrate_schedule_enrollment.sql

# 4. 第三批：成绩组成
mysql -uroot -p edu_platform < sql/migrate_grade_components.sql

# 5. 第四批：学业预警
mysql -uroot -p edu_platform < sql/migrate_academic_warning.sql

# 6. 回填 seed 数据
mysql -uroot -p edu_platform < sql/seed_course_data.sql
mysql -uroot -p edu_platform < sql/seed-all-data.sql
mysql -uroot -p edu_platform < sql/seed-test-data.sql
```

## 关键设计约定

1. class_id 命名规则：course_id × 100 + 序号（如 9101 课程的第一个班 = 910101）
2. 最终成绩计算：final_score = Σ(各组成项得分 × 权重)，由服务层计算后回填 tb_enrollment
3. 学业预警触发：final_score < 60 时系统自动生成 tb_academic_warning 记录
