# 项目当前开发目标

版本：v1.0  
日期：2026-07-09  
适用对象：后端开发对话、前端开发对话、项目成员协作

## 1. 当前阶段目标

当前阶段目标是完成在线教育辅助教学系统的第一条前后端闭环：

```text
管理员登录 -> 获取当前用户 -> 管理员用户管理 -> 前端通过网关调用后端
```

本阶段不开发课程、选课、论坛、作业和 AI 业务，只确保用户与权限基础稳定可用。

## 2. 项目目录边界

```text
springclode-alibaba-hello/
├── edu-backend/       # 后端 Maven 聚合工程
├── edu-frontend/      # 前端 Vue 工程
├── ChatGPTdocs/       # 协作文档、接口契约、设计文档
├── docker-compose.yml # MySQL + Nacos 开发环境
└── init.sql           # 数据库初始化脚本
```

后端模块位于：

```text
edu-backend/
├── edu-common/
├── edu-api/
├── edu-gateway/
├── edu-user-service/
├── edu-course-service/
├── edu-enrollment-service/
├── edu-interaction-service/
├── edu-assessment-service/
└── edu-ai-service/
```

## 3. 后端当前任务

后端对话负责：

- 维护 `edu-backend`
- 完成 `edu-common` 公共能力
- 完成 `edu-user-service` 用户服务
- 完成 `edu-gateway` 到用户服务的转发联调
- 更新 `ChatGPTdocs/API接口契约.md`

后端本阶段必须保证：

- `./mvnw -q -DskipTests compile` 可以通过
- `edu-user-service` 可以连接 MySQL
- `edu-user-service` 可以注册到 Nacos
- `edu-gateway` 可以通过服务名转发到 `edu-user-service`

## 4. 前端当前任务

前端对话负责：

- 维护 `edu-frontend`
- 初始化 Vue 3 + Vite + Element Plus
- 配置 Axios 基础请求地址
- 实现登录页
- 实现基础布局
- 实现当前用户信息读取
- 实现管理员用户管理页

前端本阶段统一通过网关访问后端：

```text
http://localhost:8080/api
```

开发阶段可临时直连用户服务调试：

```text
http://localhost:8010/api
```

但最终页面代码应以网关为准。

## 5. 基础设施

开发环境依赖 Docker 启动：

```bash
docker compose up -d
```

当前端口：

| 服务 | 地址 |
| --- | --- |
| MySQL | `127.0.0.1:3306` |
| Nacos 控制台 | `http://localhost:18848/nacos` |

后端服务使用的 Nacos 地址：

```text
127.0.0.1:18848
```

## 6. 默认账号

`edu-user-service` 启动后，如果 `tb_user` 为空，会自动创建默认管理员：

```text
username: admin
password: admin123
role: ADMIN
```

## 7. 协作规则

1. 前端对话不要修改 `edu-backend` 下的后端业务代码。
2. 后端对话不要修改 `edu-frontend` 下的页面代码。
3. 两个对话都可以读取 `ChatGPTdocs`。
4. 谁修改接口，谁必须同步更新 `ChatGPTdocs/API接口契约.md`。
5. 每次开始新对话时，先要求 Codex 阅读本文档和接口契约。
6. 每次阶段完成后，优先保证项目可以编译或启动，而不是继续堆功能。

## 8. 当前完成标准

本阶段完成标准：

- Docker 中 MySQL 和 Nacos 可以启动
- `edu-user-service` 可以启动
- `edu-gateway` 可以启动
- 可以通过网关登录：

```http
POST http://localhost:8080/api/users/login
```

- 前端登录页可以调用上述接口并保存 Token
- 管理员可以看到用户列表，并新增、编辑、删除用户
