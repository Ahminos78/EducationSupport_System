## 4. 用户与注册

登录接口：

```http
POST /api/users/login
```

注册接口：

```http
POST /api/users/register
```

注册规则：

- 只允许注册学生和教师
- `role=1` 表示学生
- `role=2` 表示教师
- 不允许通过公开注册接口注册管理员
- 管理员账号只能由系统初始化或管理员在用户管理中创建

默认管理员：

```text
username: admin
password: admin123
role: ADMIN
```

## 5. MyBatis Plus 状态

当前已从原 MyBatis 逐步整理为 MyBatis Plus。

已完成：

- 各核心实体已使用 `@TableName`
- 主键使用 `@TableId(type = IdType.AUTO)`
- 有逻辑删除字段的实体使用 `@TableLogic`
- Mapper 继承 `BaseMapper<T>`
- 删除了和 `BaseMapper.insert(T)` 同签名的自定义 `insert`
- 清理了旧的 `mybatis-spring-boot-starter` 依赖管理
- 删除旧实体：
  - `CourseSelection`
  - `ForumPost`
  - `ForumReply`

保留：

- 复杂联表查询仍使用注解 SQL
- 部分复杂更新仍使用自定义 `@Update`

这是当前阶段可接受的混合模式。


1. 确认服务已经重启
2. 确认不是旧进程占用端口

## 12. 协作注意事项

- 每次新一轮开发前，建议先阅读：
  - `ChatGPTdocs/新一轮开发上下文.md`
  - `ChatGPTdocs/API接口契约.md`
  - `ChatGPTdocs/项目当前开发目标.md`
- 后端修改接口后，必须同步更新 `ChatGPTdocs/API接口契约.md`
- 前端默认通过网关调用后端，不直接写死单个业务服务端口
- `logs/dev/*.log` 是运行产物，不建议提交
- 修改 `edu-common` 后，所有依赖它的业务服务都要重启
- 当前项目仍处于开发阶段，优先保证能编译、能启动、能联调，再继续扩展功能
