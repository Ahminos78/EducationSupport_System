## 1 引言

### 1.1 目的

本说明书在需求规格说明书的基础上，进一步细化系统各模块的功能逻辑、数据库设计、接口设计、微服务架构设计及数据交互流程，指导系统开发人员进行编码实现与单元测试，确保系统开发与用户需求一致。

### 1.2 范围

本系统设计说明书的服务对象包括高校学生、任课教师和学校管理员。系统支持学生从课程浏览、选课申请、作业提交、在线考试到AI智能答疑的完整学习流程，教师可进行课程管理、作业布置与批改、考试组卷与评分，管理员可进行用户管理、选课审批与系统维护，实现多角色协作与教学全过程管理。系统引入基于RAG（检索增强生成）技术的AI智能问答助手，结合DeepSeek大语言模型与Chroma向量数据库，为学生提供基于课程知识库的智能答疑服务。

## 2 设计概述

本系统采用前后端分离的微服务架构模式，强调模块解耦、服务自治、数据一致性和高可维护性。前端基于 Vue 3 框架与 Element Plus UI 组件库实现交互界面，支持响应式布局与多端兼容；后端使用 Spring Boot 3.2.0 构建核心服务，采用 Spring Cloud 微服务体系，通过 Nacos 实现服务注册与发现，Spring Cloud Gateway 统一管理API入口，遵循 RESTful API 规范，统一请求响应格式。

系统以 MySQL 8.0 作为主数据库，各微服务共享同一数据库实例但逻辑隔离，使用 MyBatis-Plus 简化数据访问与映射逻辑。系统引入 Spring AI 框架集成 DeepSeek 大语言模型与 SiliconFlow 嵌入模型，结合 Chroma 向量数据库实现基于 RAG 技术的AI智能问答功能，为学生提供基于课程知识库的智能答疑服务。

整体系统逻辑层划分明确，采用标准的网关层、控制器层、业务逻辑层、数据访问层四层架构，各层之间通过接口契约进行调用，各微服务之间通过 Nacos 服务发现进行通信，增强可扩展性与测试友好性。

系统支持 Web 端访问，涵盖学生、教师、管理员三类角色，提供课程管理、选课管理、作业布置与批改、在线考试与自动评分、课程讨论、AI智能问答等功能，支持快捷登录、通知提醒、多角色切换及权限审计等扩展特性。

## 3 系统技术架构

在线教育辅助系统采用标准的"前后端分离 + 微服务架构"技术模式，整体架构具备良好的解耦性、扩展性和高并发处理能力。系统采用如下技术构建：

### 3.1 前端技术栈

- **Vue 3**：构建用户界面，支持组件化、响应式数据绑定、Composition API；
- **Element Plus**：UI组件库，提供表单、表格、分页、弹窗、菜单导航等常用交互元素；
- **Axios**：用于异步发起 HTTP 请求，与后端 API 交互；
- **Vue Router**：前端路由管理，支持嵌套路由、路由守卫、动态路由加载；
- **Pinia**：Vue 3 官方推荐的状态管理库，管理用户登录状态与全局数据；
- **Vite**：新一代前端构建工具，支持热模块替换（HMR），提升开发体验；
- 支持响应式设计，适配桌面与移动终端。

### 3.2 后端技术栈

- **Spring Boot 3.2.0**：作为核心开发框架，简化配置、集成常用组件；
- **Spring Cloud 2023.0.0 + Spring Cloud Alibaba 2022.0.0.0**：微服务治理框架，提供服务发现、配置中心、网关路由；
- **Nacos**：服务注册与发现中心，同时支持动态配置管理；
- **Spring Cloud Gateway**：API网关，统一入口路由、鉴权过滤、负载均衡；
- **MyBatis-Plus 3.5.16**：简化数据库访问层，提升开发效率；
- **Spring AI 1.0.0-M6**：AI集成框架，统一接入大语言模型与向量数据库；
- 使用 **MySQL 8.0** 存储业务数据；
- 表结构符合 3NF 规范，分模块设计；
- 引入主外键约束与唯一索引保障数据一致性；
- 使用 **Lombok** 简化实体类代码；
- 使用 **Knife4j 4.5.0** 生成 RESTful API 文档。

### 3.3 缓存与服务发现技术

系统采用多层缓存机制来提升访问速度和系统性能：

- **客户端缓存**：前端基于 Vue 3，使用 LocalStorage 和 SessionStorage 存储用户偏好、临时数据、登录Token。静态资源配置浏览器缓存和打包压缩，提升页面加载速度。
- **服务注册与发现（Nacos）**：各微服务启动时向Nacos注册自身实例，消费者通过服务名动态获取实例列表，支持健康检查与故障自动摘除，保障微服务间通信的高可用性。
- **API网关缓存（Gateway）**：Gateway统一拦截请求，完成JWT鉴权后转发至下游服务，减少各服务重复鉴权的开销，同时支持跨域配置统一管理。

## 4 系统概要设计

### 4.1 体系结构设计

通过C/S结构和B/S结构的比较分析，本文确定系统的架构采用B/S架构。系统采用微服务架构，将整体业务拆分为多个独立服务，通过API网关统一对外暴露接口。主要分为以下层次：

- **数据展示层（Web层）**：Vue 3前端应用，负责用户界面渲染与交互事件处理；
- **API网关层**：Spring Cloud Gateway，负责请求路由、JWT鉴权、跨域处理、负载均衡；
- **业务服务层**：包含用户服务、课程服务、选课服务、互动服务、考试服务、AI服务六个微服务，各自处理独立的业务逻辑；
- **数据访问层**：MyBatis-Plus提供的数据访问接口，封装数据库操作逻辑；
- **数据持久层**：MySQL 8.0关系型数据库，存储所有业务数据；
- **向量存储层**：Chroma向量数据库，存储知识库文档的嵌入向量，支持语义检索；
- **服务注册中心**：Nacos，管理所有微服务的注册、发现与健康检查。

各微服务之间通过Nacos服务发现进行通信，前端请求统一经过Gateway转发至对应服务。系统的总体架构如图4.1所示。

### 4.3 功能模块概要设计

#### 4.3.1 注册登录模块

注册登录模块是系统的安全入口，负责用户身份认证与权限控制。

用户注册时，可根据提示在文本框内输入用户名、昵称、手机号、密码进行注册，系统会对密码进行BCrypt加密存储。用户登录时，输入用户名和密码进行身份验证，验证通过后系统签发JWT令牌返回给前端，后续请求均携带该令牌进行身份识别。系统还提供快捷登录功能，预置管理员、教师和学生的测试账号，方便开发调试阶段快速体验各角色功能。

角色分为三类：学生（role=1）、教师（role=2）、管理员（role=3），不同角色登录后进入不同的系统界面，拥有不同的功能权限。

#### 4.3.2 课程管理模块

课程管理模块分为前台浏览和后台管理两部分。

在前台，学生可以浏览已发布的课程列表，支持按课程名称关键词实时搜索，支持按课程性质（必修/选修/通识/公选）和课程分类进行筛选。点击课程卡片可进入课程详情页，查看课程描述、授课教师、学分、已选人数等信息。课程列表支持分页展示，默认每页12条记录。

在后台，教师可以创建新课程、编辑课程信息、上传课程封面、设置课程状态（上线/下线）。管理员拥有所有课程的管理权限，可以查看所有课程并进行编辑和状态管理。课程信息包括课程代码、课程名称、课程描述、课程封面、最大选课人数、已选人数、学分、所属学院、课程性质、课程标签等。

#### 4.3.3 选课模块

选课模块实现学生选课申请与教师审批的完整流程。

学生在课程详情页点击"申请选课"按钮，填写选课申请理由后提交，系统自动检查课程是否已满员以及学生是否已申请过该课程。选课状态分为：待审核（PENDING）、已通过（APPROVED）、已退课（DROPPED）、已拒绝（REJECTED）。

教师可以在选课管理页面查看某一课程的所有选课申请，按状态筛选，对待审核的申请进行通过或拒绝操作。通过选课时系统自动增加课程已选人数，移除学生时自动减少已选人数，保证数据一致性。学生也可以主动退课。

管理员可以查看所有课程的选课数据，进行全局的选课管理。

#### 4.3.4 作业与考试模块

作业与考试模块是教学评价的核心，包含作业管理、作业提交与批改、在线考试与自动评分四个子模块。

**作业管理**：教师可以创建作业，设置作业标题、描述、满分值、开始时间、截止时间、是否允许迟交，以及上传附件（支持最大20MB文件）。作业状态分为草稿、已发布、已关闭。教师可以编辑作业信息、管理附件、关闭作业。

**作业提交与批改**：学生在截止时间前提交作业，支持在线文本提交和附件上传。每位学生对同一作业只能提交一次，但可在截止前更新。教师查看所有学生的提交记录，对提交进行评分和评语反馈。

**在线考试与自动评分**：教师可以创建考试，设置考试标题、时间范围、满分值、考试时长、允许答题次数，并逐题录入考试题目。题目类型分为选择题（type=0）和填空题（type=1），选择题支持自动评分。学生在考试时间内开始答题，提交后系统自动判定选择题的正确性并计算得分。教师可以查看所有学生的答题记录，对自动评分结果进行复核和补充评语。

系统还提供**自动组卷功能**，教师输入课程关键词，系统根据预置题目模板库自动生成一套包含选择题和填空题的试卷。

#### 4.3.5 课程互动模块

课程互动模块提供课程讨论区功能，支持话题发布与回复。

学生和教师在已选课/已授课的课程中可以发布讨论话题，其他学生和教师可以回复。话题支持分页展示，教师和管理员可以控制话题的显示与隐藏状态。讨论内容支持编辑和删除操作，删除采用逻辑删除机制。

讨论区的权限控制基于选课关系：只有已选课的学生和课程的授课教师可以查看和参与课程讨论。

#### 4.3.6 AI智能问答模块

AI智能问答模块是本系统的特色功能，基于RAG（检索增强生成）技术，集成DeepSeek大语言模型与Chroma向量数据库，为学生提供基于课程知识库的智能答疑服务。

该模块的工作流程为：

1. **知识库构建**：教师或管理员将课程相关文档（支持PDF、DOCX、TXT格式）上传至知识库，系统通过Apache POI和PDFBox解析文档内容，使用滑动窗口算法将长文本切分为适当长度的文本块（默认500字，重叠50字），通过SiliconFlow嵌入模型（BAAI/bge-large-zh-v1.5）将文本块转换为1024维向量，存入Chroma向量数据库。
2. **语义检索**：学生提出问题时，系统将问题文本通过嵌入模型转换为查询向量，在Chroma中进行余弦相似度检索，返回最相关的文本块（默认Top-4）。
3. **回答生成**：系统将检索到的相关文本块作为上下文，连同学生的问题一起组装为Prompt，调用DeepSeek大语言模型（deepseek-v4-pro）生成回答。AI被要求仅基于知识库内容回答问题，如果知识库中没有相关信息则如实告知。

前端以浮动助手按钮的形式呈现AI问答入口，点击后弹出对话窗口，支持实时对话、显示参考来源和相关度评分。

#### 4.3.7 用户管理模块

用户管理模块面向管理员，提供对系统所有用户的管理功能。

### 4.5 接口设计

接口设计是在线教育辅助系统设计与实现的重要环节，它定义了系统内部各微服务之间以及系统与前端之间的交互方式。本节将详细介绍系统中的接口设计。

#### 4.5.1 用户接口

用户接口是用户与系统进行交互的界面，它直接影响到用户的使用体验。在本系统中，用户接口主要包括以下几个部分：

- **注册登录接口**：用于实现用户的注册和登录功能，登录成功后返回JWT令牌，前端存储令牌用于后续请求的鉴权。
- **个人信息接口**：用于查看和修改当前用户的个人信息，包括昵称、邮箱、手机号，以及修改密码。
- **课程浏览接口**：用于展示课程列表和课程详情，支持关键词搜索、分类筛选和分页查询。
- **作业提交接口**：用于学生提交作业内容和附件，以及教师批改作业。
- **在线考试接口**：用于学生开始考试、提交答案，以及查看考试结果和评分。
- **讨论互动接口**：用于发布讨论话题、回复讨论、查看讨论列表。
- **AI问答接口**：用于向AI助手提问并获取基于知识库的回答，以及构建和查询知识库。

#### 4.5.2 外部接口

外部接口是系统与外部环境进行交互的接口，它包括了系统可能需要的数据来源和数据去向。在本系统中，外部接口主要包括以下几个部分：

- **DeepSeek API接口**：通过Spring AI框架调用DeepSeek大语言模型（deepseek-v4-pro），用于AI智能问答的回答生成。
- **SiliconFlow Embedding API接口**：通过Spring AI框架调用SiliconFlow的BAAI/bge-large-zh-v1.5嵌入模型，用于将文本转换为向量表示。
- **Chroma向量数据库接口**：与Chroma向量数据库（localhost:8000）进行通信，执行向量的存储、检索和管理操作。

通过以上接口设计，本系统可以实现用户与系统之间的交互，以及系统与外部AI服务和向量数据库之间的数据交换，为学生提供智能化的学习辅助服务。

#### 4.5.3 内部接口

**内部模块间关系**

在后端微服务中，各服务之间通过Nacos服务发现进行通信，通过Gateway统一对外暴露接口。各服务的职责划分如下：

- 用户服务（:8010）：负责用户注册、登录、信息管理，其他服务通过网关转发调用用户信息接口验证身份。
- 课程服务（:8020）：负责课程的增删改查，为选课服务和互动服务提供课程信息。
- 选课服务（:8030）：负责选课申请与审批，与课程服务联动更新选课人数。
- 互动服务（:8040）：负责课程讨论区管理，与课程服务联动验证课程访问权限。
- 考试服务（:8050）：负责作业、考试、提交的全生命周期管理，支持文件附件的上传下载。
- AI服务（:8060）：负责知识库构建、语义检索和AI问答，独立运行，不直接调用其他服务。

**接口数据描述**

接口的数据统一使用JSON格式进行定义。响应数据采用统一的Result封装格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

请求数据根据具体业务需求定义，包含必要的请求参数和认证信息。每个接口均通过Knife4j自动生成OpenAPI文档，包含接口的用途、请求数据格式、响应数据格式、状态码含义等信息。

通过合理的设计和详细的描述，接口设计将极大地提升系统的可维护性和扩展性，为系统的稳定运行提供坚实的保障。

#### 5.1.1 注册登录模块

注册登录模块设计出来，是为了保证系统使用的安全性，便于管理系统的使用用户，以及收集有效数据。

用户注册登录时序图如图5.1所示。用户在前端输入登录信息后，前端通过Axios向Gateway发起POST请求（/api/users/login），Gateway进行JWT鉴权校验后转发至用户服务，用户服务查询数据库验证身份，验证通过后签发JWT令牌返回前端，前端将令牌存储至LocalStorage，后续请求通过Authorization请求头携带令牌。

```
用户 ──> 前端页面：输入用户名/密码
前端页面 ──> Gateway：POST /api/users/login {username, password}
Gateway ──> 用户服务：转发请求
用户服务 ──> MySQL：查询用户
MySQL ──> 用户服务：返回用户记录
用户服务 ──> 用户服务：BCrypt密码校验
用户服务 ──> 用户服务：签发JWT令牌
用户服务 ──> Gateway：返回 {code, token, user}
Gateway ──> 前端页面：返回响应
前端页面 ──> 前端页面：存储Token至LocalStorage
前端页面 ──> 用户：跳转至首页
```

#### 5.1.4 AI智能问答模块

AI智能问答模块是本系统的特色创新功能，旨在为学生提供基于课程知识库的智能答疑服务。

AI问答时序图如图5.4所示。学生在AI对话窗口输入问题后，系统首先将问题通过SiliconFlow嵌入模型转换为查询向量，然后在Chroma向量数据库中进行语义检索，获取最相关的文本块，最后将相关文本块作为上下文连同问题一起发送给DeepSeek大语言模型，生成回答返回给学生。

```
学生 ──> 前端AI助手：输入问题
前端 ──> Gateway：POST /api/ai/chat {question, topK}
Gateway ──> AI服务：转发请求
AI服务 ──> SiliconFlow API：问题文本 → 查询向量
SiliconFlow API ──> AI服务：返回向量
AI服务 ──> Chroma：语义检索（向量相似度匹配）
Chroma ──> AI服务：返回Top-K相关文本块
AI服务 ──> AI服务：组装Prompt（上下文 + 问题）
AI服务 ──> DeepSeek API：发送Prompt生成回答
DeepSeek API ──> AI服务：返回生成的回答
AI服务 ──> Gateway：返回 {answer, sources, model}
Gateway ──> 前端：返回响应
前端 ──> 学生：展示回答内容和参考来源
```
