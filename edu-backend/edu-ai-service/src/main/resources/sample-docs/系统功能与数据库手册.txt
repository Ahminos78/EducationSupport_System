# 在线教育辅助教学系统 — 功能与数据库完整手册

## 一、系统概述

在线教育辅助教学系统（EducationSupport_System）是一个基于 Spring Cloud 微服务架构的教育平台，支持学生、教师、管理员三种角色。系统提供课程管理、选课审批、作业发布与批改、在线考试、课程论坛、AI智能问答等核心功能。

技术架构：
- 后端：Spring Boot 3.2.0 + JDK 21 + Spring Cloud Alibaba + Nacos + MySQL 8.0 + MyBatis-Plus
- 前端：Vue 3 + Vite + Element Plus + Pinia + Vue Router
- AI模块：Spring AI + DeepSeek大语言模型 + Chroma向量数据库 + RAG检索增强生成
- 基础设施：Docker Compose（MySQL、Nacos、Chroma容器）

用户角色：
- 学生（role=1）：浏览课程、选课、提交作业、参加考试、参与论坛讨论、使用AI问答
- 教师（role=2）：创建课程、发布作业与考试、批改作业、管理选课、管理课程论坛
- 管理员（role=3）：管理所有用户、管理所有课程、管理选课审批、拥有教师权限

## 二、数据库设计

系统数据库名为 edu_platform，包含13张核心业务表，均使用 InnoDB 引擎和 utf8mb4 字符集。

### 2.1 用户相关表

tb_user 用户表：存储所有系统用户的基本信息。字段包括用户ID（自增主键）、登录用户名（唯一）、密码哈希（BCrypt加密）、昵称、角色（1=学生,2=教师,3=管理员）、创建时间、更新时间、逻辑删除标记。所有密码测试账号统一为123456。

tb_student_profile 学生档案表：以user_id为主键关联用户表，存储学号（唯一）、真实姓名、学院、专业、班级、年级、入学年份等学籍信息。

tb_teacher_profile 教师档案表：以user_id为主键关联用户表，存储工号（唯一）、真实姓名、所属学院或部门、职称等职业信息。

### 2.2 课程相关表

tb_course 课程表：记录课程基本信息。字段包括课程ID、课程号、授课教师ID（关联用户表）、课程名称、课程简介、封面图地址、最大选课人数（默认100）、当前已选人数、学分、开课单位、课程性质（必修/选修/通识/个性课程）、课程标签、教学班个数、课程状态（0=下架,1=正常）、逻辑删除标记。

tb_enrollment 选课表：记录学生选课信息。字段包括选课记录ID、课程ID、学生ID、选课状态（0=待审核,1=已通过,2=已退课,4=已拒绝）、选课申请说明、审核意见、申请时间、审核时间。课程ID和学生ID有联合唯一约束，防止重复选课。

### 2.3 作业相关表

tb_assignment 作业表：教师发布的作业。字段包括作业ID、课程ID、教师ID、作业标题、作业说明、满分（默认100）、开始时间、截止时间、是否允许延期提交（0=不允许,1=允许）、作业状态（0=草稿,1=已发布,2=已截止）、发布时间、逻辑删除标记。

tb_submission 作业提交表：学生的作业提交记录。字段包括提交ID、作业ID、学生ID、提交内容、附件地址、提交状态（0=暂存,1=已提交,2=已撤回）、批改状态（0=待批改,1=已批改）、得分、教师评语、AI辅助评语、提交时间、批改时间。作业ID和学生ID有联合唯一约束。

tb_assignment_attachment 教师作业附件表：教师上传的作业附件。字段包括附件ID、作业ID、原始文件名、服务器存储文件名、文件类型、文件大小、上传教师ID、上传时间。

tb_submission_attachment 学生提交附件表：学生作业提交的附件。字段包括附件ID、提交记录ID、原始文件名、服务器存储文件名、文件类型、文件大小、上传时间。

### 2.4 考试相关表

tb_exam 考试表：教师发布的考试。字段包括考试ID、课程ID、教师ID、考试标题、考试说明、开始时间、结束时间、满分（默认100）、考试时长（分钟，默认60）、允许提交次数（默认1）、考试状态（0=草稿,1=已发布,2=已结束）、逻辑删除标记。

tb_question 考试题目表：考试中的具体题目。字段包括题目ID、考试ID、题型（0=选择题,1=填空题）、题目内容、选项（JSON数组格式，仅选择题）、正确答案、题目分值（默认10）、题目顺序。

tb_exam_attempt 考试参与记录表：学生的考试作答记录。字段包括参与记录ID、考试ID、学生ID、作答内容（LONGTEXT）、参与状态（0=进行中,1=已提交,2=已批改）、考试得分、教师评语、开始时间、交卷时间、批改时间。考试ID和学生ID有联合唯一约束。

### 2.5 论坛相关表

tb_discussion 课程讨论表：支持主题帖和回复的树形讨论。字段包括讨论ID、课程ID、父讨论ID（NULL表示主题帖，非空表示回复）、作者ID、主题帖标题（回复可为空）、讨论内容、状态（0=隐藏,1=正常）、逻辑删除标记。父讨论ID自关联引用主键ID，支持多层嵌套回复。

## 三、系统功能模块

### 3.1 用户认证与管理

登录功能：用户通过用户名和密码登录，系统返回JWT令牌用于后续请求鉴权。
注册功能：支持学生和教师自主注册，注册时需选择角色。
快捷登录：提供演示账号快捷登录，包含管理员、教师、学生三种角色的测试账号。
个人中心：用户可查看和编辑个人资料，包括昵称、头像等基本信息。
修改密码：用户可修改自己的登录密码，需输入原密码验证。
用户管理（管理员）：管理员可分页查看所有用户，支持按关键词和角色筛选，可创建、编辑、删除用户。

### 3.2 课程管理

创建课程：教师可创建新课程，填写课程名称、简介、学分、最大选课人数、课程性质等信息。
课程列表：所有用户可浏览课程列表，支持按关键词搜索、按课程性质和类别筛选，支持分页。
课程详情：查看课程详细信息，包括课程简介、授课教师、选课人数、已布置的作业和考试列表。
课程状态：教师和管理员可上架或下架课程。
课程删除：教师和管理员可逻辑删除课程。

### 3.3 选课系统

学生选课：学生可浏览可选课程列表，提交选课申请，填写选课理由。
选课审批：教师和管理员可审核选课申请，批准或拒绝学生选课。
选课管理：教师和管理员可查看课程的选课学生列表，支持按状态筛选。
退课功能：学生可主动退课，教师也可将学生从课程中移除。
选课统计：课程表记录当前已选人数，与最大选课人数配合进行容量控制。

### 3.4 作业系统

发布作业：教师可创建作业，设置标题、说明、满分、开始时间、截止时间，支持上传作业附件。
作业列表：课程详情页展示该课程的所有作业，按时间排序。
提交作业：学生在截止时间前提交作业内容，可上传附件。每位学生对每个作业只能有一条提交记录。
作业批改：教师查看学生提交列表，对每份作业进行评分和评语反馈。系统支持AI辅助评语功能。
作业状态管理：作业有草稿、已发布、已截止三种状态，教师可手动切换。

### 3.5 考试系统

创建考试：教师可创建考试，设置标题、时间范围、时长、满分、允许提交次数。支持一次性创建考试并批量添加题目。
题目管理：支持选择题（含ABCD选项）和填空题两种题型。教师可添加、编辑、删除题目，设置每题分值和顺序。
自动出题：系统支持根据课程信息自动生成题目。
在线答题：学生在考试时间范围内开始作答，系统记录开始时间。答题界面展示所有题目，学生填写答案后提交。
自动批改：系统在学生提交时自动比对答案进行评分。
试卷批阅：教师可查看学生答卷详情，进行人工批改和评语反馈。
考试记录：系统记录每位学生的每次考试参与状态、得分、提交时间。

### 3.6 课程论坛

发表主题：学生和教师可在课程论坛中发表讨论主题帖。
回复讨论：其他用户可对主题帖进行回复，支持多层嵌套。
内容管理：作者可编辑和删除自己的帖子。教师和管理员可设置帖子的显示或隐藏状态。
分页浏览：论坛支持按课程分页浏览主题帖列表，查看主题详情和回复列表。

### 3.7 AI智能问答

知识库构建：管理员或教师可将教学文档（支持PDF、Word、TXT格式）上传至知识库。系统自动解析文档内容，将文本按固定大小分块（默认500字符，50字符重叠），通过SiliconFlow的BAAI/bge-large-zh-v1.5模型将文本转换为1024维向量，存入Chroma向量数据库。

语义检索：用户输入问题后，系统将问题向量化，在Chroma中进行余弦相似度搜索，返回最相关的文档块。

RAG问答：系统将检索到的文档块作为参考资料，与用户问题一起组装为Prompt，发送给DeepSeek大语言模型（deepseek-v4-pro），生成基于知识库的精准回答。回答中包含引用来源和相关度评分。

前端集成：前端页面右下角悬浮AI问答按钮，点击打开对话框，支持快捷问题模板，展示对话历史和参考来源。

## 四、API接口总览

系统通过Spring Cloud Gateway（端口8080）统一入口，路由到各个微服务：
- 用户服务（edu-user-service，端口8010）：/api/users/*
- 课程服务（edu-course-service，端口8020）：/api/courses/*
- 选课服务（edu-enrollment-service，端口8030）：/api/enrollments/*
- 互动服务（edu-interaction-service，端口8040）：/api/interactions/*
- 考评服务（edu-assessment-service，端口8050）：/api/assessments/*
- AI服务（edu-ai-service，端口8060）：/api/ai/* 和 /api/knowledge/*

所有接口统一返回格式：{ "code": 200, "message": "success", "data": {} }
